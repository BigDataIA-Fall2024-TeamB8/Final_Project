Background

TED Talks are a great way to learn since they provide insightful thoughts from speakers on a range of subjects. Without particular searches, users may have trouble finding the correct content, and there are no tools to compare ideas from different lectures or customize insights. Additionally, the platform is devoid of interactive features like conversational browsing and methods for converting insights into practical actions. Because of these restrictions, viewers find it challenging to properly interact with and implement the information presented in TED Talks, highlighting the need for a more customized and interactive platform.

Objective

Our project aims to develop an AI-powered platform that:

  1. Enabling viewers to quickly locate relevant TED Talks with AI-powered interactive applications.
  2. Utilizing Retrieval-Augmented Generation (RAG) in a chatbot assistant to provide particular responses and personalized recommendations.
  3. Helps with taking notes, making mental maps, and creating research notes and playbooks

Scope

  • Data Sources: TED metadata and transcripts via TED API, YouTube Transcript API, Whisper (speech-to-text).
  • Technologies: FastAPI for backend APIs, React.js for frontend, Airflow for ETL pipelines, Pinecone for vector storage, OpenAI APIs for NLP tasks.
  • Deliverables:
  • Intelligent TED Talk search and discovery.
  • A Q&A chatbot assistant using RAG (Retrieval-Augmented Generation).
  • Tools for creating notes, mind maps, and playbooks.

Stakeholders

  • Learners and Researchers: Students, educators, and professionals.
  • Content Enthusiasts: TED users seeking deeper knowledge.
  • Organizations: Teams using TED Talks for training and development.

In the modern world, TED lectures provide a variety of information, but viewers frequently struggle to select necessary lectures, customize the insights to suit their own requirements, and apply the knowledge to real-world situations. This undermines the platform's effect and use.

Current Challenges:

  • It can be difficult for people to find speeches that are relevant to more general or complex queries because TED's search engine relies on exact keywords.
  • TED Talk insights are not customized to each user's learning objectives or circumstances, which limits their impact in real-world situations.
  • There are no integrated tools available to users for turning interactive ideas into organized research outputs like mind maps, useful playbooks, or notes.

Opportunities:

  • Use of Retrieval-Augmented Generation (RAG) and conversational querying inspired by AI to improve finding contents and insight extraction.
  • Allow users to adapt and customize TED Talk insights according to their own requirements and goals.
  • By offering interactive resources for note-taking, comparison analysis, and the development of actionable research, you can transform passive use of data into active learning and application.

We employed an agentic architecture that allows our platform to be purpose driven and allow collaboration between various endpoints. Below is a detailed description of the agents:

Core Agents

  1. Search Endpoint
  • Function: Process user queries to find the most relevant TED Talks and return them
  • Logic: Uses embeddings stored in Pinecone to perform similarity searches based on the query
  • Example:
  • Input: "Talks on leadership and emotional intelligence."
  • Output: Ranked list of talks with links and summaries.
  1. Retrieval Augmented Generation Agent (RAG):
  • Function: Answer user questions by utilizing RAG
  • Logic:
  • Fetches relevant transcript snippets using RAG.
  • Uses OpenAI API to generate relevant naturalized responses.
  • Example:
  • Input: "What does Richard Dawkins say about growing up in the universe?"
  • Output: Detailed answer with references to the talk and additional insights.
  1. Web Search Agent:
  • Function: Perform web search for topics to gain additional context
  • Logic:
  • Uses Tavily API to perform google searches
  • Retrieves relevant links
  • Uses OpenAI API to summarize learnings
  • Example:
  • Input: "The speaker mentioned Julius Caesar. How did he rise to power?"
  • Output: Detailed answer about Caesar's history
  1. Talk Comparison Agent
  • Function: Compare multiple talks transcripts based on ideas and themes
  • Logic:
  • Identifies common and opposing themes by retrieving relevant transcripts
  • Provides detailed comparison response highlighting features of studied talks
  • Example:
  • Input: "Compare Bill Gates and Jim Snabe's talks on leadership."
  • Output: Highlighted differences and similarities.
  1. Note-Taking Endpoint
  • Function: Enable users to take notes while watching talks.
  • Logic:
  • Links notes to specific timestamps in the transcript.
  • Organizes notes by themes or user-defined categories.
  • Uses OpenAI LLM for note generation
  • Example:
  • User highlights a transcript section, and the agent converts it into a note.
  1. Mind Map Endpoint
  • Function: Create visual representations of a talk's ideas and their interconnections.
  • Logic:
  • Parses transcripts to identify core topics and minor ideas.
  • Generates interactive graphs using Cytoscape.js.
  • Example:
  • Input: "Generate a mind map for Simon Sinek's leadership talk."
  • Output: A graph showing nodes for key ideas like "Trust" and "Collaboration."
  1. Research Note Agent
  • Task: Consolidate user notes, user contexts, mind maps, and chatbot responses into a structured research note or playbook.
  • Intelligence:
  • Organizes content into sections (e.g., summary, themes,views).
  • Export as PDF or Markdown.
  • Example Interaction:
  • User requests a playbook on "Generate a report on the leadership talk but for a student."
